<font size='5'>**Dense Hierarchical Dual-Task Network for Landslide Detection in Yunnan Province**</font>

Haozhuo Huang, [Junchuan Yu](https://github.com/JunchuanYu)☨, Daqing Ge, Shufang Tian, Ling Zhang, Laidian Xi2, Qiong Wu1, Changhong Hou3, Tingyan Fu

☨corresponding author: yujunchuan@mail.cgs.gov.cn

本仓库为论文 **Dense Hierarchical Dual-Task Network for Landslide Detection in Yunnan Province** 的配套源代码库，主要用于复现文中提出的 DHD-Net 模型、三阶段训练流程、消融实验以及大幅面 GeoTIFF 预测流程。

## Overview

云南省地形破碎、山地面积占比高、降雨充沛，滑坡灾害频发。针对复杂地质环境下传统局部检测方法难以支撑大范围自动识别的问题，论文提出了 **DHD-Net**。该方法结合多源遥感数据与 Stacking-InSAR 结果，完成滑坡隐患的高精度检测。

根据论文描述，DHD-Net 的核心思路包括：

- 使用多源遥感数据联合建模，包括 InSAR 形变相位图、DEM、Bing Maps 光学影像和积雪覆盖信息。
- 采用双任务结构，将形变检测与类别判别解耦，降低不同任务之间的特征干扰。
- 在解码器中引入 DFA 模块，缓解双编码器带来的信息冗余，增强特征融合效果。
- 面向大范围滑坡隐患识别任务，兼顾检测完整性与类别区分能力。

论文实验结果表明，DHD-Net 在自建数据集上取得了：

- F1-Score: `77.55%`
- Recall: `80.24%`
- Precision: `75.03%`
- IoU: `63.33%`

并在云南省范围内最终识别出 `4059` 处滑坡隐患。

## Method Summary

从代码实现看，DHD-Net 由三个部分组成：

1. 分类网络：使用多源输入进行类别特征提取。
2. 分割网络：使用形变相关输入进行目标区域提取。
3. 双任务融合网络：加载前两阶段编码器权重并冻结，再通过融合解码器完成最终预测。

## Training

训练入口为 `Train.ipynb`，整体采用三阶段训练策略：

### Stage 1: Classification Network

- 使用 `Classification_Network` 训练分类分支。
- 默认忽略文件名前缀为 `0` 的样本。
- 优化器为 `AdamW`。
- 默认学习率为 `0.001`，`batch_size` 为 `8`。
- 监控指标为 `MRecall`。

### Stage 2: Segmentation Network

- 使用 `Segmentation_Network` 训练分割分支。
- 默认分割类别数为 `2`。
- 监控指标为 `Binary IoU`。

### Stage 3: Dual-Task Fusion Network

- 构建 `Dual_Task_Fusion_Network`。
- 加载前两阶段编码器权重。
- 冻结 `stageMS_*` 与 `stageSAR_*` 编码器参数。
- 训练融合解码部分，最终保存权重到 `./Model_Weight/DHD-Net.pkl`。
- 监控指标为 `F1_1`。

Notebook 中默认超参数包括：

```python
class_num = 6
seg_class_num = 2
lr = 0.001
beta1 = 0.9
beta2 = 0.999
weight_decay = 0.01
class_epochs = 150
seg_epochs = 150
dual_epochs = 150
batch_size = 8
```

训练时还实现了基于滑动窗口均值的早停机制。

## Inference

预测入口为 `Predict.ipynb`。

默认流程如下：

- 从 `--data_dir` 读取待预测的 `.tif` 数据。
- 使用滑窗方式逐块推理。
- 默认窗口大小 `img_size = 128`，步长 `stride = 128`。
- 结果输出到 `./Data/Predictions`。
- 输出文件保留原始 GeoTIFF 的投影与地理参考信息。

默认权重路径：

```text
./Model_Weight/DHD-Net.pkl
```

## Ablation Study

消融实验入口为 `Ablation_Study.ipynb`，当前仓库包含两组论文核心消融：

- `Without Dual-Task`

Notebook 中给出的测试集对比结果如下：
