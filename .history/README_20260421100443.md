## 概述

本仓库为论文配套源代码库，主要用于复现文中提出的 DHD-Net 模型、三阶段训练流程、消融实验以及预测流程。

根据论文描述，DHD-Net 的核心思路包括：

- 使用多源遥感数据联合建模，包括 InSAR 形变相位图、DEM、Bing Maps 光学影像和积雪覆盖信息。
- 采用双任务结构，将形变检测与类别判别解耦，降低不同任务之间的特征干扰。
- 在解码器中引入 DFA 模块，缓解双编码器带来的信息冗余，增强特征融合效果。
- 面向大范围滑坡隐患识别任务，兼顾检测完整性与类别区分能力。

论文实验结果表明，DHD-Net 在自建数据集上取得了：

- F1 分数：`77.55%`
- 召回率：`80.24%`
- 精确率：`75.03%`
- 交并比：`63.33%`

并在云南省范围内最终识别出 `4059` 处滑坡隐患。

## 方法概述

从代码实现看，DHD-Net 由三个部分组成：

1. 分类网络：使用多源输入进行类别特征提取。
2. 分割网络：使用形变相关输入进行目标区域提取。
3. 双任务融合网络：加载前两阶段编码器权重并冻结，再通过融合解码器完成最终预测。

## 训练

训练入口为 `Train.ipynb`，整体采用三阶段训练策略：

### 阶段一：分类网络

- 使用 `Classification_Network` 训练分类分支。
- 默认忽略文件名前缀为 `0` 的样本。
- 优化器为 `AdamW`。
- 默认学习率为 `0.001`，`batch_size` 为 `8`。
- 监控指标为 `MRecall`。

### 阶段二：分割网络

- 使用 `Segmentation_Network` 训练分割分支。
- 默认分割类别数为 `2`。
- 监控指标为 `Binary IoU`。

### 阶段三：双任务融合网络

- 构建 `Dual_Task_Fusion_Network`。
- 加载前两阶段编码器权重。
- 冻结 `stageMS_*` 与 `stageSAR_*` 编码器参数。
- 训练融合解码部分，最终保存权重到 `./Model_Weight/DHD-Net.pkl`。
- 监控指标为 `F1_1`。

Notebook 中默认超参数包括：

```python
class_num = 6
seg_class_num = 2
lr = 0.001
beta1 = 0.9
beta2 = 0.999
weight_decay = 0.01
class_epochs = 150
seg_epochs = 150
dual_epochs = 150
batch_size = 8
```

训练时还实现了基于滑动窗口均值的早停机制。

## 预测

预测入口为 `Predict.ipynb`。

默认流程如下：

- 从 `--data_dir` 读取待预测的 `.tif` 数据。
- 使用滑窗方式逐块推理。
- 默认窗口大小 `img_size = 128`，步长 `stride = 128`。
- 结果输出到 `./Data/Predictions`。
- 输出文件保留原始 GeoTIFF 的投影与地理参考信息。

默认权重路径：

```text
./Model_Weight/DHD-Net.pkl
```

## 消融实验

消融实验入口为 `Ablation_Study.ipynb`，当前仓库包含两组论文核心消融：

- `Without Dual-Task`
- `Without DFA`

Notebook 中给出的测试集对比结果如下：

| 模型 | F1 分数 (%) | 召回率 (%) | 精确率 (%) | 交并比 (%) |
| --- | ---: | ---: | ---: | ---: |
| UNet++ | 70.33 | 70.85 | 69.82 | 54.24 |
| Without Dual-Task | 75.17 | 73.23 | 77.21 | 60.21 |
| Without DFA | 75.25 | 75.51 | 75.00 | 60.32 |
| DHD-Net | 77.55 | 80.24 | 75.03 | 63.33 |
